# SPEC-UI-IMPROVE-001: 페이지 UI/GUI 개선

> **Status**: draft
> **Created**: 2026-02-27
> **Priority**: High
> **Category**: UI/UX Enhancement

---

## 1. 개요

카카오톡 프로필 UI 레퍼런스와 경쟁 분석 보고서를 기반으로, Namecard 서비스의 주요 페이지 UI/GUI를 현대적이고 세련된 디자인으로 개선합니다.

### 참고 자료
- **레퍼런스 이미지**: 카카오톡 프로필 화면 (원형 아바타, 수평 액션 바, 깔끔한 리스트, 미디어 갤러리)
- **경쟁 분석**: `_AEC/competitive-analysis-report.md` (15개 경쟁사 비교, SWOT 분석)

### 핵심 인사이트 (레퍼런스 이미지)
| 요소 | 카카오톡 레퍼런스 | 현재 Namecard |
|------|-------------------|---------------|
| 프로필 아바타 | 대형 원형 (120px+), 중앙 배치 | 소형 (96px), 좌측 배치 |
| 이름 표시 | 중앙, 인증 뱃지, 굵은 폰트 | 좌측 정렬, 뱃지 없음 |
| 액션 바 | 수평 4개 아이콘 (프로필/검색/알림/옵션) | 없음 |
| 리스트 항목 | 아이콘 + 제목 + 부제목, 깔끔한 구분 | 없음 |
| 미디어 갤러리 | 하단 그리드, 탭 전환 | 카드 그리드 (기본적) |
| 전체 느낌 | 미니멀, 여백 활용, 직관적 | 기능적이나 세련미 부족 |

## 2. 개선 범위

### 2.1 프로필 페이지 리디자인 (`/profile/[id]`)

#### EARS 요구사항

**REQ-PROF-001**: 프로필 헤더 리디자인
> WHEN 사용자가 프로필 페이지에 접근하면,
> THE SYSTEM SHALL 카카오톡 스타일의 중앙 정렬 프로필 헤더를 표시한다.

개선 내용:
```
현재 레이아웃:
┌──────────────────────────────────┐
│ [< Gallery]         [Namecard]   │  ← 네비게이션
├──────────────────────────────────┤
│ [Avatar 96px] 이름               │  ← 좌측 정렬
│               바이오              │
│               카드 3 · 좋아요 12  │
│               [테마 뱃지들]       │
├──────────────────────────────────┤
│ [카드] [카드] [카드] [카드]       │  ← 그리드
└──────────────────────────────────┘

개선 레이아웃:
┌──────────────────────────────────┐
│ [<]                    [공유] [⋯] │  ← 미니멀 네비게이션
├──────────────────────────────────┤
│                                   │
│         [Avatar 120px]            │  ← 중앙 대형 아바타
│           이름 ✓                  │  ← 중앙 + 뱃지
│         @handle · 직함            │  ← 부가 정보
│                                   │
│    [편집]  [QR]  [공유]  [설정]   │  ← 수평 액션 바
│                                   │
├──── 통계 바 ─────────────────────┤
│   카드 12   |  좋아요 48  | 조회 156│  ← 통계 섹션
├──────────────────────────────────┤
│ 바이오                            │
│ "디자인으로 세상을 바꿉니다"      │
│                                   │
│ 🔗 instagram · linkedin · email  │  ← 소셜 링크 아이콘
├──────────────────────────────────┤
│  [전체] [포켓몬] [해리포터] [타로] │  ← 테마 필터 탭
├──────────────────────────────────┤
│ [카드] [카드] [카드]              │  ← 카드 그리드
│ [카드] [카드] [카드]              │
└──────────────────────────────────┘
```

상세 변경:
- 아바타: 96px → 120px, 중앙 배치, 그림자 + 테두리 링 효과
- 이름: 중앙 정렬, `text-2xl font-bold`, 선택적 인증 뱃지
- 수평 액션 바: 4개 아이콘 버튼 (편집/QR코드/공유/설정)
- 통계 섹션: 카드 수, 좋아요 합계, 조회수 - 균등 3분할
- 바이오: 별도 섹션으로 분리, 소셜 링크 아이콘 포함
- 테마 필터 탭: 보유 카드 기준 테마 필터링

---

**REQ-PROF-002**: 프로필 액션 바
> WHEN 프로필 페이지가 로드되면,
> THE SYSTEM SHALL 수평 정렬된 4개 퀵 액션 버튼을 표시한다.

| 액션 | 아이콘 | 본인 프로필 | 타인 프로필 |
|------|--------|------------|------------|
| 편집/팔로우 | ✏️ / 👤+ | 프로필 편집 이동 | 팔로우 (향후) |
| QR 코드 | 📱 | QR 모달 표시 | QR 모달 표시 |
| 공유 | 🔗 | 프로필 링크 공유 | 프로필 링크 공유 |
| 설정/더보기 | ⚙️ / ⋯ | 설정 페이지 이동 | 신고/차단 (향후) |

---

**REQ-PROF-003**: 카드 테마 필터 탭
> WHEN 프로필에 여러 테마의 카드가 있으면,
> THE SYSTEM SHALL 테마별 필터 탭을 표시하여 카드를 필터링할 수 있게 한다.

- "전체" 탭 (기본 선택)
- 보유 테마만 탭으로 표시
- 각 탭에 카드 수 뱃지 표시
- 스크롤 가능한 수평 탭 (테마 많을 경우)

### 2.2 카드 갤러리 페이지 개선 (`/cards`)

#### EARS 요구사항

**REQ-GAL-001**: 갤러리 헤더 리디자인
> WHEN 사용자가 갤러리 페이지에 접근하면,
> THE SYSTEM SHALL 검색, 필터, 정렬 기능이 통합된 세련된 헤더를 표시한다.

개선 내용:
```
현재:
┌──────────────────────────────────┐
│ 명함 갤러리 (123장)               │
│ 모든 명함을 둘러보세요             │
├──────────────────────────────────┤
│ [피드] [이벤트]                   │
├──────────────────────────────────┤
│ 테마필터: [전체▼]  정렬: [최신▼]  │

개선:
┌──────────────────────────────────┐
│ 갤러리                     [🔍]  │  ← 깔끔한 타이틀 + 검색
├──────────────────────────────────┤
│ [전체] [인기] [최신] [이벤트]     │  ← 통합 탭 네비게이션
├──────────────────────────────────┤
│ 🏷 [클래식] [포켓몬] [타로] [+]  │  ← 테마 칩 필터 (수평 스크롤)
├──────────────────────────────────┤
│ [카드] [카드] [카드]              │
```

상세 변경:
- 검색 아이콘 → 확장형 검색 바 (이름/해시태그 검색)
- 탭 통합: 피드/이벤트 → 전체/인기/최신/이벤트 4탭
- 테마 칩 필터: 수평 스크롤 가능한 칩 형태 필터
- "인기" 탭: 좋아요 수 기준 정렬

---

**REQ-GAL-002**: 카드 썸네일 개선
> WHEN 카드 썸네일이 갤러리에 표시되면,
> THE SYSTEM SHALL 호버 시 추가 정보와 인터랙션을 제공한다.

개선 내용:
- 호버 오버레이: 반투명 배경 + 좋아요/북마크 버튼 표시
- 호버 시 카드 제작자 정보 표시 (아바타 + 이름)
- 좋아요 카운트 상시 표시 (우측 하단)
- 롱프레스(모바일): 빠른 미리보기 모달
- 카드 비율 유지: 29:45 → 더 큰 썸네일 (모바일에서 가독성 향상)

---

**REQ-GAL-003**: 검색 기능 추가
> WHEN 사용자가 검색 아이콘을 클릭하면,
> THE SYSTEM SHALL 이름, 해시태그, 이벤트명으로 카드를 검색할 수 있는 검색 바를 표시한다.

- 실시간 검색 (debounce 300ms)
- 검색 대상: 카드 이름, 해시태그, 이벤트명
- 검색 결과 하이라이트
- 최근 검색어 저장 (로컬스토리지)

### 2.3 카드 상세 페이지 개선 (`/cards/[id]`)

#### EARS 요구사항

**REQ-CARD-001**: 카드 상세 뷰 리디자인
> WHEN 사용자가 카드 상세 페이지에 접근하면,
> THE SYSTEM SHALL 몰입형 카드 뷰와 풍부한 인터랙션을 제공한다.

개선 내용:
```
현재:
┌──────────────────────────────────┐
│        [카드 프리뷰]              │
│                                   │
│  [좋아요] [북마크] [vCard]        │
│        카드 이름                  │
└──────────────────────────────────┘

개선:
┌──────────────────────────────────┐
│ [<]              [⋯ 더보기]       │  ← 미니멀 헤더
├──────────────────────────────────┤
│                                   │
│        ┌───────────────┐          │
│        │               │          │
│        │   카드 앞면    │          │  ← 대형 카드 (터치로 뒤집기)
│        │               │          │
│        └───────────────┘          │
│     탭하여 뒤집기 · 1/2           │
│                                   │
├──────────────────────────────────┤
│  [❤️ 12]  [🔖]  [📤 공유]  [💾]  │  ← 액션 바 (하단 고정)
├──────────────────────────────────┤
│  👤 제작자 이름                   │  ← 제작자 정보
│  📅 2026-02-15                   │
│  🏷 #디자인 #개발자 #창업         │  ← 해시태그
├──────────────────────────────────┤
│  비슷한 카드                      │  ← 추천 섹션
│  [카드] [카드] [카드] [카드]      │
└──────────────────────────────────┘
```

상세 변경:
- 카드 크기 확대: 화면 폭 80% 활용 (max-w-sm → max-w-md)
- 터치/클릭 플립: 탭하여 앞뒤 전환 (3D 회전 애니메이션)
- 하단 고정 액션 바: 좋아요, 북마크, 공유, 다운로드
- 공유 버튼 확장: 카카오톡, 링크 복사, QR 코드, SNS 공유
- 제작자 정보 섹션: 아바타, 이름 (프로필 링크), 날짜
- 해시태그 표시: 클릭 시 해당 태그 갤러리 필터링
- "비슷한 카드" 추천: 같은 테마/이벤트 카드 수평 스크롤

---

**REQ-CARD-002**: 공유 바텀시트 개선
> WHEN 사용자가 공유 버튼을 클릭하면,
> THE SYSTEM SHALL 다양한 공유 옵션을 포함한 바텀시트를 표시한다.

공유 옵션 (우선순위순):
1. 카카오톡 공유 (커스텀 템플릿)
2. 링크 복사
3. QR 코드 표시
4. PNG 다운로드 (앞면/뒷면/양면)
5. vCard 다운로드
6. Twitter/Instagram/LinkedIn 공유

### 2.4 대시보드 개선 (`/dashboard`)

#### EARS 요구사항

**REQ-DASH-001**: 대시보드 리디자인
> WHEN 인증된 사용자가 대시보드에 접근하면,
> THE SYSTEM SHALL 프로필 요약과 주요 기능에 빠르게 접근할 수 있는 대시보드를 표시한다.

개선 내용:
```
현재:
┌──────────────────────────────────┐
│ [Logo]              [UserMenu]   │
│ 안녕하세요, 홍길동님               │
│ [새 명함] [설정] [북마크] [관리자] │
│ 내 요청 현황                      │
│ [요청 리스트...]                  │
└──────────────────────────────────┘

개선:
┌──────────────────────────────────┐
│ [Logo]              [🔔] [👤]   │  ← 알림 + 프로필
├──────────────────────────────────┤
│      [Avatar 80px]               │
│      홍길동                       │  ← 프로필 요약 카드
│      카드 3장 · 좋아요 12         │
│      [프로필 보기]                │
├──────────────────────────────────┤
│  ┌────┐ ┌────┐ ┌────┐ ┌────┐   │
│  │ 새  │ │ 내  │ │ 북  │ │ 설  │  ← 퀵 액션 그리드
│  │ 명함│ │ 카드│ │ 마크│ │ 정  │     (2x2 그리드)
│  └────┘ └────┘ └────┘ └────┘   │
├──────────────────────────────────┤
│ 내 카드 미리보기                  │  ← 수평 스크롤 카드
│ [카드1] [카드2] [카드3] →        │
├──────────────────────────────────┤
│ 최근 활동                         │
│ · 카드 "개발자 명함" 전달 완료     │  ← 활동 피드
│ · 좋아요 +3 받음                  │
│ · 새 카드 요청 접수               │
└──────────────────────────────────┘
```

상세 변경:
- 프로필 요약 카드: 아바타 + 이름 + 통계 + 프로필 링크
- 퀵 액션: 2x2 또는 4열 그리드 (아이콘 + 텍스트)
- 내 카드 미리보기: 수평 스크롤 카드 리스트 (최근 3-5장)
- 최근 활동 피드: 상태 변경, 좋아요, 북마크 등 활동 내역
- 알림 아이콘: 새 알림 있으면 뱃지 표시

### 2.5 모바일 네비게이션 개선

#### EARS 요구사항

**REQ-NAV-001**: 모바일 하단 네비게이션
> WHEN 모바일 화면(768px 미만)에서 앱을 사용하면,
> THE SYSTEM SHALL 하단 고정 네비게이션 바를 표시한다.

```
┌──────────────────────────────────┐
│                                   │
│         (페이지 콘텐츠)            │
│                                   │
├──────────────────────────────────┤
│  🏠    📇    ➕    🔖    👤     │  ← 하단 네비게이션
│  홈    갤러리  만들기 북마크  내정보 │
└──────────────────────────────────┘
```

탭 구성:
| 탭 | 아이콘 | 경로 | 설명 |
|----|--------|------|------|
| 홈 | Home | `/dashboard` | 대시보드 |
| 갤러리 | Grid | `/cards` | 명함 갤러리 |
| 만들기 | Plus | `/create` | 새 명함 생성 (강조) |
| 북마크 | Bookmark | `/dashboard/bookmarks` | 저장한 카드 |
| 내정보 | User | `/profile/me` | 내 프로필 |

- "만들기" 버튼: 중앙에 강조 (크기 크게, 메인 컬러)
- 현재 탭 활성화 표시: 아이콘 색상 + 라벨 강조
- 데스크탑에서는 숨김 (기존 상단 네비게이션 유지)

### 2.6 전반적 디자인 시스템 개선

#### EARS 요구사항

**REQ-DESIGN-001**: 디자인 토큰 개선
> THE SYSTEM SHALL 일관된 디자인 시스템을 적용하여 시각적 품질을 향상시킨다.

| 항목 | 현재 | 개선 |
|------|------|------|
| 모서리 | 0px (sharp corners) | 8-16px (rounded, 부드러운 느낌) |
| 그림자 | 최소 사용 | 3단계 그림자 (sm/md/lg) |
| 여백 | 일관성 부족 | 8px 그리드 시스템 (8/16/24/32/48) |
| 색상 | 다크 네이비 위주 | 밝은 배경 + 포인트 컬러 강화 |
| 타이포그래피 | 3폰트 혼용 | 계층 체계 정리 (h1-h6 + body + caption) |
| 아이콘 | 혼재 (텍스트 + 이모지) | 통일된 아이콘 세트 (Lucide Icons) |
| 애니메이션 | 기본적 | 미세 인터랙션 추가 (마이크로 애니메이션) |

**REQ-DESIGN-002**: 밝은 모드 기본 + 다크 모드 지원
> THE SYSTEM SHALL 기본 밝은 모드에서 세련된 디자인을 제공하고,
> 선택적으로 다크 모드를 지원한다.

- 배경: `#FFFFFF` → `#F8F9FA` (미세한 회색톤)
- 카드 배경: `#FFFFFF` (순백)
- 텍스트: `#1A1A1A` (소프트 블랙)
- 포인트: `#020912` (현재 Primary 유지)

## 5. 기술 고려사항

### 사용 기술
- **아이콘**: Lucide React (일관된 아이콘 세트)
- **애니메이션**: CSS transitions + Framer Motion (카드 플립, 페이지 전환)
- **반응형**: Tailwind CSS breakpoints (sm/md/lg/xl)
- **검색**: 클라이언트 사이드 필터링 + API 엔드포인트

### 성능 고려
- 이미지 lazy loading 유지
- CSS-only 애니메이션 우선 (JS 애니메이션 최소화)
- 하단 네비게이션: position fixed + z-index 관리
- 검색: debounce 적용, 서버 사이드 검색 API

### 호환성
- 모바일 Safari, Chrome (iOS/Android)
- 데스크탑 Chrome, Firefox, Safari, Edge
- 최소 화면 너비: 320px
