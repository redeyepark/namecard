# SPEC-THEME-001: 인수 기준

## 테스트 시나리오

### AC-001: 테마 시스템 기본 지원 (REQ-U-001)

**Given** 명함 편집 애플리케이션이 로드된 상태
**When** 테마 선택기를 확인할 때
**Then** 최소 "Classic"과 "Pokemon" 2개의 테마 옵션이 표시된다

---

### AC-002: 기존 카드 하위 호환성 (REQ-U-002)

**Given** localStorage에 `theme` 필드가 없는 기존 카드 데이터가 저장된 상태
**When** 애플리케이션을 로드할 때
**Then** 기존 카드가 Classic 테마로 정상 렌더링된다
**And** 기존의 배경색, 텍스트 색상, 콘텐츠가 변경 없이 표시된다

---

### AC-003: 오리지널 SVG 아이콘 (REQ-U-003)

**Given** Pokemon 테마가 선택된 상태
**When** 타입 아이콘이 카드에 표시될 때
**Then** 모든 아이콘은 자체 디자인된 SVG이며 포켓몬 상표/이미지가 포함되지 않는다

---

### AC-004: 테마 선택기 제공 (REQ-E-001)

**Given** 사용자가 명함 편집 페이지에 접근한 상태
**When** 에디터 패널을 확인할 때
**Then** Classic과 Pokemon 테마를 선택할 수 있는 테마 선택기가 표시된다
**And** 선택기는 `role="radiogroup"` 접근성 속성을 포함한다

---

### AC-005: Pokemon 테마 앞면 렌더링 (REQ-E-002)

**Given** 사용자가 Pokemon 테마를 선택하고 "Fire" 타입을 설정한 상태
**When** 카드 앞면 미리보기를 확인할 때
**Then** 카드에 `#FF6B35` 색상의 4px solid 테두리 프레임이 표시된다
**And** 좌상단에 사용자 이름이 네임 플레이트 스타일로 표시된다
**And** 우상단에 EXP 스탯 값이 표시된다
**And** 스탯 옆에 Fire 타입 SVG 아이콘이 표시된다
**And** 중앙 영역에 프로필 사진이 프레임 처리되어 표시된다
**And** 하단에 해시태그가 스타일링된 섹션에 표시된다

---

### AC-006: Pokemon 테마 뒷면 렌더링 (REQ-E-003)

**Given** 사용자가 Pokemon 테마를 선택한 상태
**When** 카드 뒷면 미리보기를 확인할 때
**Then** blue-to-purple 그라데이션 배경이 적용된다
**And** CSS로 생성된 장식적 소용돌이 패턴이 표시된다
**And** 사용자의 풀네임과 직함이 표시된다
**And** 하단에 소셜 링크가 스타일링되어 표시된다
**And** 해시태그가 표시된다

---

### AC-007: 타입 색상 즉시 반영 (REQ-E-004)

**Given** Pokemon 테마가 선택된 상태에서 현재 타입이 "Fire"인 경우
**When** 사용자가 타입을 "Water"로 변경할 때
**Then** 카드 프레임 색상이 `#FF6B35`에서 `#3B82F6`으로 즉시 변경된다
**And** 페이지 새로고침 없이 실시간으로 반영된다

---

### AC-008: 테마 전환 시 데이터 보존 (REQ-E-005)

**Given** Classic 테마에서 이름 "홍길동", 직함 "개발자", 해시태그 ["#React", "#Next"], 소셜 링크 1개, 프로필 사진이 설정된 상태
**When** Pokemon 테마로 전환할 때
**Then** 이름 "홍길동"이 보존된다
**And** 직함 "개발자"가 보존된다
**And** 해시태그 ["#React", "#Next"]가 보존된다
**And** 소셜 링크가 보존된다
**And** 프로필 사진이 보존된다

**Given** Pokemon 테마에서 동일한 데이터가 설정된 상태
**When** Classic 테마로 다시 전환할 때
**Then** 모든 콘텐츠 데이터가 동일하게 보존된다

---

### AC-009: EXP 값 설정 및 표시 (REQ-E-006)

**Given** Pokemon 테마가 선택된 상태
**When** 사용자가 EXP 입력 필드에 "500"을 입력할 때
**Then** 카드 우상단에 "EXP 500"이 표시된다

**Given** Pokemon 테마가 선택된 상태
**When** 사용자가 EXP에 1000 이상을 입력할 때
**Then** 값이 999로 제한된다

**Given** Pokemon 테마가 선택된 상태
**When** 사용자가 EXP에 음수를 입력할 때
**Then** 값이 0으로 제한된다

**Given** Pokemon 테마가 처음 선택된 상태
**When** EXP 기본값을 확인할 때
**Then** "EXP 100"이 기본값으로 표시된다

---

### AC-010: theme 필드 미존재 시 기본 처리 (REQ-S-001)

**Given** `{ front: {...}, back: {...} }` 형태의 카드 데이터 (theme 필드 없음)
**When** 시스템이 카드를 렌더링할 때
**Then** Classic 테마 레이아웃으로 렌더링된다
**And** 기존 `CardFront`, `CardBack` 렌더링과 시각적으로 동일하다

---

### AC-011: Pokemon 테마 PNG 내보내기 (REQ-S-002)

**Given** Pokemon 테마가 선택되고 Fire 타입, EXP 300이 설정된 상태
**When** 카드를 PNG로 내보내기할 때
**Then** 생성된 PNG 이미지에 타입 색상 프레임이 포함된다
**And** EXP 스탯 텍스트가 포함된다
**And** 타입 SVG 아이콘이 포함된다
**And** 뒷면 PNG에 그라데이션 배경과 소용돌이 패턴이 포함된다

---

### AC-012: 테마별 에디터 옵션 (REQ-S-003)

**Given** Pokemon 테마가 선택된 상태
**When** 에디터 패널을 확인할 때
**Then** 타입 선택기 (7가지 타입 그리드)가 표시된다
**And** EXP 숫자 입력 필드가 표시된다

**Given** Classic 테마가 선택된 상태
**When** 에디터 패널을 확인할 때
**Then** 타입 선택기와 EXP 입력 필드가 표시되지 않는다

---

### AC-013: 디자인 원본성 검증 (REQ-N-001)

**Given** Pokemon 테마의 전체 시각적 요소를 검토할 때
**When** 디자인 감사를 수행할 때
**Then** 포켓몬 브랜드 로고, 상표가 사용되지 않았다
**And** 타입 아이콘은 모두 자체 디자인된 SVG이다
**And** 카드 레이아웃은 트레이딩 카드에서 "영감을 받은" 오리지널 디자인이다

---

### AC-014: 테마 전환 시 데이터 미삭제 (REQ-N-002)

**Given** 사용자가 모든 필드를 입력한 상태
**When** 테마를 여러 번 반복 전환할 때 (Classic -> Pokemon -> Classic -> Pokemon)
**Then** 매 전환마다 콘텐츠 데이터가 완전히 보존된다
**And** 프로필 사진이 삭제되지 않는다
**And** 소셜 링크가 삭제되지 않는다

---

### AC-015: 범용 테마 인터페이스 (REQ-O-001, Optional)

**Given** 개발자가 새로운 테마를 추가하려는 상태
**When** 테마 설정 인터페이스를 확인할 때
**Then** `ThemeConfig` 인터페이스에 맞춰 새 테마를 등록할 수 있는 구조가 존재한다

---

## 추가 검증 시나리오

### CSV 일괄 업로드 호환성

**Given** theme 컬럼이 없는 기존 CSV 파일
**When** 관리자가 일괄 업로드를 실행할 때
**Then** 모든 카드가 Classic 테마로 생성된다
**And** 업로드 프로세스에 에러가 발생하지 않는다

### 관리자 미리보기

**Given** Pokemon 테마로 생성된 카드 요청이 존재하는 상태
**When** 관리자가 요청 상세 페이지에서 미리보기를 확인할 때
**Then** Pokemon 테마 스타일(프레임, 타입 아이콘, EXP)이 정확히 표시된다

### 대시보드 미리보기

**Given** Pokemon 테마로 확정된 카드가 존재하는 상태
**When** 사용자가 대시보드에서 자신의 확정 카드를 확인할 때
**Then** Pokemon 테마 스타일이 정확히 표시된다

### 세션 지속성

**Given** 사용자가 Pokemon 테마를 선택하고 Fire 타입, EXP 250을 설정한 상태
**When** 브라우저를 닫고 다시 열 때
**Then** 테마 선택(Pokemon), 타입(Fire), EXP(250)가 모두 복원된다

## Quality Gate

| 항목                  | 기준                                     |
| --------------------- | ---------------------------------------- |
| 하위 호환성           | 기존 Classic 카드 렌더링 변경 없음        |
| PNG 내보내기          | 두 테마 모두 정확한 이미지 생성           |
| 접근성                | 테마 선택기 키보드 탐색 가능              |
| 디자인 원본성         | 포켓몬 상표/저작권 이미지 미사용          |
| 데이터 보존           | 테마 전환 시 콘텐츠 100% 보존            |
| 세션 지속성           | localStorage 테마/옵션 복원 동작          |

---

## Definition of Done

1. Classic, Pokemon 2개 테마 모두 정상 렌더링된다
2. 테마 전환 시 사용자 데이터가 보존된다
3. PNG 내보내기가 두 테마 모두에서 동작한다
4. 기존 카드(theme 필드 없음)가 Classic으로 정상 표시된다
5. CSV 일괄 업로드가 기존과 동일하게 동작한다
6. 관리자/대시보드 미리보기가 테마별로 정확히 표시된다
7. 포켓몬 상표/저작권 이미지가 사용되지 않았다
