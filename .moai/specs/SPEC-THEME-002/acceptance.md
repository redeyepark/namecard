# SPEC-THEME-002: 인수 기준

## 테스트 시나리오

### AC-001: 리스트 박스에 5개 테마 표시 (REQ-U-001)

**Given** 관리자가 테마 관리 페이지(`/admin/themes`)에 접근한 상태
**When** 리스트 박스를 확인할 때
**Then** Classic, Pokemon, Hearthstone, Harry Potter, Tarot 5개 테마가 모두 목록에 표시된다
**And** 각 테마 항목이 클릭 가능한 상태이다

---

### AC-002: 한 번에 하나의 테마 미리보기 (REQ-U-002)

**Given** 리스트 박스에서 테마가 선택된 상태
**When** 미리보기 영역을 확인할 때
**Then** 선택된 테마 하나의 카드만 대형으로 표시된다
**And** 다른 테마의 카드가 동시에 표시되지 않는다

---

### AC-003: 디자인 시스템 준수 (REQ-U-003)

**Given** 테마 관리 페이지가 렌더링된 상태
**When** 전체 UI를 검사할 때
**Then** 배경/텍스트 색상이 Deep navy(#020912) + off-white(#fcfcfc) 기반이다
**And** 모든 모서리가 Sharp corners(0px border-radius)이다
**And** 모든 클릭 가능 요소의 터치 타겟이 최소 44px이다

---

### AC-004: 2컬럼 레이아웃 표시 (REQ-TL-001)

**Given** 관리자가 데스크톱 브라우저(1024px 이상)에서 테마 관리 페이지에 접근한 상태
**When** 페이지 레이아웃을 확인할 때
**Then** 좌측에 테마 리스트 박스(사이드바)가 표시된다
**And** 우측에 대형 미리보기 영역과 편집 패널이 표시된다

---

### AC-005: 데스크톱 컬럼 너비 (REQ-TL-002)

**Given** 데스크톱(1024px 이상) 화면에서 페이지가 표시된 상태
**When** 레이아웃 치수를 확인할 때
**Then** 좌측 리스트 박스가 약 280px 고정 너비이다
**And** 우측 미리보기 영역이 나머지 공간을 차지한다

---

### AC-006: 모바일 레이아웃 변환 (REQ-TL-003)

**Given** 모바일(1024px 미만) 화면에서 페이지가 표시된 상태
**When** 레이아웃을 확인할 때
**Then** 사이드바 리스트 박스 대신 드롭다운 선택기 또는 수평 탭바가 상단에 표시된다
**And** 미리보기 영역이 전체 너비로 아래에 표시된다

---

### AC-007: 리스트 박스 항목 구성 (REQ-LB-001)

**Given** 리스트 박스가 렌더링된 상태
**When** 각 테마 항목을 확인할 때
**Then** 테마 컬러 인디케이터(예: Classic=#020912, Pokemon=#EED171)가 표시된다
**And** 테마 이름이 한국어로 표시된다 (예: "클래식", "포켓몬")
**And** 의뢰 건수 배지가 "N건" 형식으로 표시된다

---

### AC-008: 리스트 박스 선택 동작 (REQ-LB-002)

**Given** Classic 테마가 선택된 상태
**When** 관리자가 "포켓몬" 항목을 클릭할 때
**Then** "포켓몬" 항목이 활성 상태(어두운 배경, 밝은 텍스트)로 변경된다
**And** "클래식" 항목이 비활성 상태로 변경된다
**And** 미리보기 영역이 Pokemon 테마 카드로 전환된다

---

### AC-009: 기본 선택 상태 (REQ-LB-003)

**Given** 관리자가 테마 관리 페이지에 최초 접근한 상태
**When** 리스트 박스의 기본 선택을 확인할 때
**Then** "클래식" 테마가 활성 상태로 표시된다
**And** 미리보기 영역에 Classic 테마 카드가 표시된다

---

### AC-010: 대형 미리보기 크기 (REQ-PV-001)

**Given** 테마가 선택된 상태
**When** 미리보기 영역의 카드를 확인할 때
**Then** 카드가 기존 그리드 미리보기보다 큰 크기(높이 400~500px 범위)로 표시된다
**And** 카드의 aspect-ratio(29:45)가 유지된다

---

### AC-011: 앞면/뒷면 토글 (REQ-PV-002)

**Given** 미리보기 영역에 테마 카드가 표시된 상태
**When** 앞면/뒷면 토글 버튼을 클릭할 때
**Then** 카드의 앞면과 뒷면이 전환된다

---

### AC-012: 실시간 미리보기 업데이트 (REQ-PV-003)

**Given** Pokemon 테마가 선택되고 미리보기에 "Electric" 타입 카드가 표시된 상태
**When** 편집 패널에서 타입을 "Fire"로 변경할 때
**Then** 미리보기의 카드가 즉시 Fire 타입 디자인으로 업데이트된다
**And** 페이지 새로고침이 발생하지 않는다

---

### AC-013: 테마별 variant 선택기 표시 - Pokemon (REQ-ED-001)

**Given** Pokemon 테마가 선택된 상태
**When** 편집 패널을 확인할 때
**Then** 7가지 타입(Fire, Water, Grass, Electric, Psychic, Steel, Normal) 칩 버튼이 표시된다
**And** 각 칩에 타입 아이콘과 이름이 포함된다

**Given** Hearthstone 테마가 선택된 상태
**When** 편집 패널을 확인할 때
**Then** 9가지 직업(Warrior, Mage, Rogue, Priest, Hunter, Paladin, Shaman, Warlock, Druid) 칩 버튼이 표시된다

**Given** Harry Potter 테마가 선택된 상태
**When** 편집 패널을 확인할 때
**Then** 4가지 기숙사(Gryffindor, Slytherin, Hufflepuff, Ravenclaw) 선택 버튼이 표시된다

**Given** Tarot 테마가 선택된 상태
**When** 편집 패널을 확인할 때
**Then** 5가지 아르카나(Major, Wands, Cups, Swords, Pentacles) 선택 버튼이 표시된다

---

### AC-014: 테마별 메타데이터 입력 (REQ-ED-002)

**Given** Pokemon 테마가 선택된 상태
**When** 편집 패널을 확인할 때
**Then** EXP 숫자 입력 필드(0-999)가 표시된다

**Given** Hearthstone 테마가 선택된 상태
**When** 편집 패널을 확인할 때
**Then** Mana(0-10), Attack(0-12), Health(1-12) 숫자 입력 필드가 표시된다

**Given** Harry Potter 테마가 선택된 상태
**When** 편집 패널을 확인할 때
**Then** Year(1-7), Spell Power(0-999) 숫자 입력 필드가 표시된다

**Given** Tarot 테마가 선택된 상태
**When** 편집 패널을 확인할 때
**Then** Card Number(0-21), Mystique(0-999) 숫자 입력 필드가 표시된다

---

### AC-015: Classic 테마 편집 패널 (REQ-ED-003)

**Given** Classic 테마가 선택된 상태
**When** 편집 패널을 확인할 때
**Then** variant 선택기가 표시되지 않는다
**And** 메타데이터 입력 필드가 표시되지 않는다
**And** 테마 설명 텍스트만 표시된다

---

### AC-016: 편집 시 즉시 업데이트 (REQ-ED-004)

**Given** Hearthstone 테마가 선택되고 Mage 직업이 적용된 상태
**When** Mana 값을 3에서 7로 변경할 때
**Then** 미리보기 카드의 마나 크리스탈 숫자가 즉시 7로 업데이트된다
**And** 페이지 새로고침 없이 변경이 반영된다

---

### AC-017: 일괄 적용 섹션 위치 (REQ-BA-001)

**Given** 관리자가 테마 관리 페이지에 접근한 상태
**When** 페이지를 스크롤할 때
**Then** 리스트 박스 + 미리보기 영역 아래에 "테마 일괄 적용" 섹션이 존재한다
**And** 기존의 상태 필터, 현재 테마 필터, 적용 대상 테마 선택, 메타데이터 옵션이 모두 표시된다

---

### AC-018: 일괄 적용 API 동작 유지 (REQ-BA-002)

**Given** 관리자가 일괄 적용을 실행할 때
**When** 적용 버튼을 클릭하고 확인할 때
**Then** `PATCH /api/admin/themes` API가 기존과 동일한 body 형식으로 호출된다
**And** 성공/실패 메시지가 기존과 동일하게 표시된다

---

### AC-019: 로컬 상태 관리 (REQ-DF-001)

**Given** 미리보기 영역에서 Pokemon 타입을 변경한 상태
**When** 브라우저 개발자 도구 네트워크 탭을 확인할 때
**Then** variant/메타데이터 변경으로 인한 API 호출이 발생하지 않는다
**And** 변경 사항은 React 컴포넌트의 로컬 상태에만 저장된다

---

### AC-020: 통계 데이터 배지 표시 (REQ-DF-002)

**Given** 페이지가 로드된 상태
**When** `GET /api/admin/themes` 응답이 수신된 후
**Then** 각 테마의 의뢰 건수가 리스트 박스 항목의 배지에 표시된다

---

### AC-021: 통계 로딩 상태 (REQ-S-001)

**Given** 페이지가 로드되고 통계 데이터를 가져오는 중인 상태
**When** 리스트 박스를 확인할 때
**Then** 배지 영역에 로딩 인디케이터(스켈레톤 또는 스피너)가 표시된다

---

### AC-022: 0건 테마 표시 (REQ-S-002)

**Given** 특정 테마의 의뢰가 0건인 상태
**When** 리스트 박스에서 해당 테마를 확인할 때
**Then** 배지에 "0건"이 표시된다
**And** 배지 자체가 숨겨지지 않는다

---

### AC-023: 테마별 편집 상태 독립 보존 (REQ-N-001)

**Given** Pokemon 테마에서 "Fire" 타입, EXP 500을 설정한 상태
**When** Hearthstone 테마로 전환했다가 다시 Pokemon 테마로 돌아올 때
**Then** Pokemon 테마의 설정이 "Fire" 타입, EXP 500으로 유지된다
**And** 이전 편집 상태가 삭제되지 않는다

---

### AC-024: 서버 상태 미변경 (REQ-N-002)

**Given** 미리보기 영역에서 다양한 테마의 variant와 메타데이터를 변경한 상태
**When** 실제 카드 데이터베이스를 확인할 때
**Then** 어떠한 카드 데이터도 변경되지 않는다
**And** 서버 API에 의도치 않은 변경 요청이 전송되지 않는다

---

### AC-025: 리스트 박스 썸네일 (REQ-O-001, Optional)

**Given** 리스트 박스가 렌더링된 상태
**When** 각 테마 항목을 확인할 때
**Then** 테마를 대표하는 축소 썸네일 또는 아이콘이 컬러 인디케이터 옆에 표시된다

---

### AC-026: 스와이프 제스처 (REQ-O-002, Optional)

**Given** 모바일 환경에서 미리보기 영역이 표시된 상태
**When** 카드를 좌우로 스와이프할 때
**Then** 앞면과 뒷면이 스와이프 방향에 따라 전환된다

## 추가 검증 시나리오

### 모든 테마 순회 검증

**Given** 관리자가 테마 관리 페이지에 접근한 상태
**When** 리스트 박스에서 5개 테마를 순서대로 클릭할 때
**Then** 각 테마의 미리보기 카드가 올바르게 렌더링된다
**And** 각 테마의 편집 패널이 해당 테마에 맞는 옵션을 표시한다
**And** 모든 전환이 에러 없이 수행된다

### variant 변경 후 테마 전환 복원

**Given** Pokemon 테마에서 "Water" 타입을 선택하고, Hearthstone 테마에서 "Warrior" 직업을 선택한 상태
**When** 다시 Pokemon 테마를 선택할 때
**Then** Pokemon의 variant가 "Water"로 유지된다

**When** 다시 Hearthstone 테마를 선택할 때
**Then** Hearthstone의 variant가 "Warrior"로 유지된다

### 일괄 적용 기능 회귀 테스트

**Given** 리스트 박스 + 미리보기 레이아웃으로 변경된 이후
**When** 하단의 일괄 적용 섹션에서 테마 적용을 실행할 때
**Then** 기존과 동일하게 필터 → 대상 선택 → 적용 → 확인 다이얼로그 플로우가 동작한다
**And** 적용 결과 메시지가 정상 표시된다
**And** 통계가 업데이트되어 리스트 박스 배지에 반영된다

### 브라우저 리사이즈 반응형

**Given** 데스크톱(1200px) 화면에서 페이지가 표시된 상태
**When** 브라우저 너비를 768px로 줄일 때
**Then** 사이드바 리스트 박스가 사라지고 드롭다운 선택기가 나타난다
**And** 미리보기 영역이 전체 너비로 확장된다

**When** 브라우저 너비를 다시 1200px로 늘릴 때
**Then** 사이드바 리스트 박스가 다시 나타난다
**And** 이전 테마 선택 상태가 유지된다

## Definition of Done

1. 5개 테마 모두 리스트 박스에서 선택할 수 있다
2. 선택된 테마의 카드가 대형(400-500px) 미리보기로 표시된다
3. 앞면/뒷면 토글이 정상 동작한다
4. 각 테마의 variant 선택기가 올바르게 표시되고 동작한다
5. 메타데이터 편집이 미리보기에 실시간 반영된다
6. Classic 테마는 variant/메타데이터 편집기가 표시되지 않는다
7. 테마 전환 시 각 테마의 편집 상태가 독립적으로 보존된다
8. 미리보기 편집이 서버 API를 호출하지 않는다
9. 기존 일괄 적용(Section C) 기능이 그대로 동작한다
10. 테마 통계가 리스트 박스 배지로 표시된다
11. 데스크톱 2컬럼 / 모바일 단일 컬럼 반응형이 동작한다
12. 프로젝트 디자인 시스템(#020912, 0px radius, 44px 터치 타겟)을 준수한다
