## 1. 모바일 하단 탭바 인수 기준

### AC-001: 하단 탭바 기본 렌더링

**Given** 인증된 사용자가 모바일 뷰포트(< 768px)에서 비관리자 페이지를 방문한 상태
**When** 페이지가 로드됨
**Then** 화면 하단에 고정된 탭바가 표시되어야 한다
**And** 탭바에는 4개 탭(홈, 커뮤니티, 마이, 프로필)이 표시되어야 한다
**And** 각 탭에는 아이콘과 한국어 라벨이 표시되어야 한다

### AC-002: 홈 탭 네비게이션

**Given** 하단 탭바가 표시된 상태
**When** 사용자가 "홈" 탭을 탭함
**Then** /cards (갤러리) 페이지로 이동해야 한다
**And** 홈 탭이 활성 상태로 표시되어야 한다

### AC-003: 커뮤니티 탭 네비게이션

**Given** 하단 탭바가 표시된 상태
**When** 사용자가 "커뮤니티" 탭을 탭함
**Then** /community/questions 페이지로 이동해야 한다
**And** 커뮤니티 탭이 활성 상태로 표시되어야 한다

### AC-004: 마이 탭 네비게이션

**Given** 하단 탭바가 표시된 상태
**When** 사용자가 "마이" 탭을 탭함
**Then** /dashboard 페이지로 이동해야 한다
**And** 마이 탭이 활성 상태로 표시되어야 한다

### AC-005: 프로필 탭 네비게이션

**Given** 사용자 ID가 "abc-123"인 인증된 사용자의 하단 탭바가 표시된 상태
**When** 사용자가 "프로필" 탭을 탭함
**Then** /profile/abc-123 (현재 사용자의 프로필) 페이지로 이동해야 한다
**And** 프로필 탭이 활성 상태로 표시되어야 한다

### AC-006: 활성 탭 표시자

**Given** 사용자가 /community/questions/some-id 페이지에 있는 상태
**When** 하단 탭바가 렌더링됨
**Then** "커뮤니티" 탭이 활성 상태(진한 아이콘 색상, 하단 밑줄)로 표시되어야 한다
**And** 나머지 탭은 비활성 상태(흐린 아이콘 색상)로 표시되어야 한다

### AC-007: 커피챗 배지 표시

**Given** 커피챗 pending 수신 건수가 3건인 상태
**When** 하단 탭바가 렌더링됨
**Then** "커뮤니티" 탭 아이콘 옆에 "3"이 표시된 배지가 나타나야 한다

### AC-008: 커피챗 배지 비표시

**Given** 커피챗 pending 수신 건수가 0건인 상태
**When** 하단 탭바가 렌더링됨
**Then** "커뮤니티" 탭에 배지가 표시되지 않아야 한다

## 2. 데스크톱 상단 네비게이션 인수 기준

### AC-010: 상단 네비게이션 기본 렌더링

**Given** 인증된 사용자가 데스크톱 뷰포트(>= 768px)에서 비관리자 페이지를 방문한 상태
**When** 페이지가 로드됨
**Then** 페이지 상단에 네비게이션 바가 표시되어야 한다
**And** 왼쪽에 "Namecard" 로고가 표시되어야 한다
**And** 갤러리, 커뮤니티 링크가 표시되어야 한다
**And** 오른쪽에 UserMenu가 표시되어야 한다

### AC-011: 활성 링크 표시

**Given** 사용자가 데스크톱에서 /cards 페이지에 있는 상태
**When** 상단 네비게이션이 렌더링됨
**Then** "갤러리" 링크에 활성 표시자(하단 밑줄)가 표시되어야 한다

### AC-012: 커뮤니티 링크 배지

**Given** 커피챗 pending 수신 건수가 5건인 상태
**When** 상단 네비게이션이 렌더링됨
**Then** "커뮤니티" 링크 옆에 "5"가 표시된 배지가 나타나야 한다

### AC-013: UserMenu 드롭다운 열기

**Given** 인증된 사용자의 상단 네비게이션이 표시된 상태
**When** 사용자가 UserMenu 영역(아바타/이름)을 클릭함
**Then** 드롭다운 메뉴가 표시되어야 한다
**And** "내 프로필", "대시보드", "북마크", "설정", "로그아웃" 항목이 표시되어야 한다

### AC-014: UserMenu 드롭다운 - 내 프로필

**Given** UserMenu 드롭다운이 열린 상태 (현재 사용자 ID: "abc-123")
**When** "내 프로필"을 클릭함
**Then** /profile/abc-123 페이지로 이동해야 한다
**And** 드롭다운이 닫혀야 한다

### AC-015: UserMenu 드롭다운 - 로그아웃

**Given** UserMenu 드롭다운이 열린 상태
**When** "로그아웃"을 클릭함
**Then** signOut()이 호출되어야 한다
**And** 사용자가 랜딩 페이지로 리다이렉트되어야 한다

### AC-016: UserMenu 드롭다운 외부 클릭

**Given** UserMenu 드롭다운이 열린 상태
**When** 드롭다운 외부 영역을 클릭함
**Then** 드롭다운이 닫혀야 한다

### AC-017: 미인증 사용자 상단 네비게이션

**Given** 미인증 사용자가 데스크톱에서 공개 페이지(/cards/some-id)를 방문한 상태
**When** 상단 네비게이션이 렌더링됨
**Then** 로고와 "로그인" 버튼만 표시되어야 한다
**And** 갤러리, 커뮤니티 링크, UserMenu는 표시되지 않아야 한다

## 3. 반응형 동작 인수 기준

### AC-020: 데스크톱에서 하단 탭바 숨김

**Given** 데스크톱 뷰포트(>= 768px)에서 페이지를 방문한 상태
**When** 페이지가 로드됨
**Then** 하단 탭바가 화면에 표시되지 않아야 한다

### AC-021: 모바일에서 상단 네비게이션 숨김

**Given** 모바일 뷰포트(< 768px)에서 페이지를 방문한 상태
**When** 페이지가 로드됨
**Then** 데스크톱 상단 네비게이션이 화면에 표시되지 않아야 한다

### AC-022: 콘텐츠 가림 방지

**Given** 모바일 뷰포트에서 긴 콘텐츠가 있는 페이지를 방문한 상태
**When** 페이지 최하단까지 스크롤함
**Then** 콘텐츠의 마지막 요소가 하단 탭바에 의해 가려지지 않아야 한다
**And** 콘텐츠 하단에 탭바 높이(64px) 이상의 여백이 존재해야 한다

### AC-023: 레이아웃 시프트 없음

**Given** 모바일 뷰포트에서 페이지를 로드하는 상태
**When** 네비게이션 컴포넌트가 렌더링됨
**Then** 콘텐츠 영역에 눈에 띄는 레이아웃 시프트가 발생하지 않아야 한다

## 4. 통합 인수 기준

### AC-030: CommunityNav와의 공존

**Given** 인증 사용자가 모바일에서 /community/questions를 방문한 상태
**When** 페이지가 로드됨
**Then** 하단 탭바의 "커뮤니티" 탭이 활성 상태로 표시되어야 한다
**And** 페이지 내부의 CommunityNav(질문/커피챗 서브탭)도 정상적으로 표시되어야 한다
**And** 두 네비게이션이 시각적으로 충돌하지 않아야 한다

### AC-031: 관리자 페이지 네비게이션 분리

**Given** 관리자 사용자가 /admin 페이지를 방문한 상태
**When** 페이지가 로드됨
**Then** GlobalNav(상단 네비게이션 + 하단 탭바)가 표시되지 않아야 한다
**And** 기존 admin/layout.tsx의 관리자 전용 네비게이션이 정상 표시되어야 한다

### AC-032: 랜딩 페이지 미인증 사용자

**Given** 미인증 사용자가 랜딩 페이지(/)를 방문한 상태
**When** 페이지가 로드됨
**Then** GlobalNav가 표시되지 않아야 한다
**And** 기존 랜딩 페이지 디자인이 그대로 유지되어야 한다

### AC-033: 랜딩 페이지 인증 사용자

**Given** 인증 사용자가 랜딩 페이지(/)를 방문한 상태
**When** 페이지가 로드됨
**Then** GlobalNav(상단 네비게이션 + 하단 탭바)가 표시되어야 한다

### AC-034: 대시보드 헤더 제거 확인

**Given** 인증 사용자가 /dashboard를 방문한 상태
**When** 페이지가 로드됨
**Then** 기존 독립 헤더(로고 + UserMenu)가 표시되지 않아야 한다
**And** GlobalNav가 유일한 네비게이션으로 표시되어야 한다
**And** 대시보드 콘텐츠가 정상적으로 표시되어야 한다

## 5. 접근성 인수 기준

### AC-040: ARIA 속성

**Given** 네비게이션 컴포넌트가 렌더링된 상태
**When** DOM을 검사함
**Then** 하단 탭바의 `<nav>` 요소에 `aria-label="주 메뉴"` 또는 유사한 라벨이 존재해야 한다
**And** 활성 탭에 `aria-current="page"` 속성이 존재해야 한다

### AC-041: 키보드 네비게이션

**Given** 데스크톱에서 상단 네비게이션이 표시된 상태
**When** 사용자가 Tab 키로 네비게이션 항목을 순회함
**Then** 로고 -> 갤러리 -> 커뮤니티 -> UserMenu 순서로 포커스가 이동해야 한다
**And** 각 항목에 `focus-visible` 스타일이 적용되어야 한다

### AC-042: UserMenu 드롭다운 키보드

**Given** UserMenu에 포커스가 있는 상태
**When** Enter 또는 Space 키를 누름
**Then** 드롭다운이 열려야 한다

**Given** UserMenu 드롭다운이 열린 상태
**When** Escape 키를 누름
**Then** 드롭다운이 닫혀야 한다
**And** 포커스가 UserMenu 토글 버튼으로 복귀해야 한다

### AC-043: 최소 터치 영역

**Given** 모바일에서 하단 탭바가 렌더링된 상태
**When** 각 탭의 터치 영역을 측정함
**Then** 모든 탭의 터치 영역이 최소 44px x 44px 이상이어야 한다

## 6. 금지 동작 인수 기준

### AC-050: 커피챗 폴링 중복 방지

**Given** /community/questions 페이지가 로드된 상태 (GlobalNav + CommunityNav 동시 렌더링)
**When** 60초 간격의 폴링 사이클이 실행됨
**Then** /api/coffee-chat/pending-count 엔드포인트가 사이클당 1회만 호출되어야 한다

### AC-051: 네비게이션 깜빡임 없음

**Given** 인증된 사용자가 페이지 간 이동하는 상태
**When** Next.js 클라이언트 사이드 네비게이션이 실행됨
**Then** GlobalNav 컴포넌트가 깜빡이거나 사라졌다 나타나지 않아야 한다

### AC-052: 콘텐츠 가림 없음

**Given** 모바일에서 모든 비관리자 페이지를 방문한 상태
**When** 페이지 최하단까지 스크롤함
**Then** 어떤 콘텐츠도 하단 탭바에 의해 가려지지 않아야 한다

## 7. 디자인 시스템 인수 기준

### AC-060: 색상 체계 준수

**Given** 네비게이션 컴포넌트가 렌더링된 상태
**When** 시각적으로 검사함
**Then** 배경색은 #fcfcfc 또는 white를 사용해야 한다
**And** 활성 요소 색상은 #020912을 사용해야 한다
**And** 비활성 요소 색상은 #020912의 40% 투명도를 사용해야 한다
**And** 배지 색상은 기존 CoffeeChatBadge와 동일한 #f59e0b를 사용해야 한다

### AC-061: border-radius 준수

**Given** 네비게이션 컴포넌트가 렌더링된 상태
**When** DOM 스타일을 검사함
**Then** 모든 네비게이션 요소에 border-radius: 0px이 적용되어야 한다 (둥근 모서리 없음)

### AC-062: 폰트 준수

**Given** 네비게이션 컴포넌트가 렌더링된 상태
**When** 텍스트 스타일을 검사함
**Then** 네비게이션 라벨과 링크에 Figtree 폰트가 적용되어야 한다
